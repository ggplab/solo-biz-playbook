# Solo Biz Playbook

> 1인 사업자·크리에이터·인디 교육자를 위한 **린캔버스 + 의사결정 원칙** 공개 플레이북.

실제로 굴리고 있는 1인 사업의 린캔버스와, 그 위에서 추출한 **재사용 가능한 원칙**을 공개합니다. 이론 튜토리얼이 아니라, 한 명의 구체 사례 + 독자가 채워 쓰는 워크북 + 재사용 규율 3층 구조입니다.

---

## 왜 만들었나

1인 사업 린캔버스는 *원리*만 있고 *실례*가 없습니다. 책에는 B2B SaaS·실리콘밸리 스타트업 사례만 있고, 한국 교육·크리에이터 맥락의 1인 사업자 케이스는 거의 공개돼 있지 않습니다.

이 레포는 그 공백을 메우려고 만들었습니다. "대충 이런 걸 채우세요"가 아니라, **실제로 한 명이 채워놓은 것**을 공개해 감을 잡게 하고, 거기서 뽑은 의사결정 규율을 따로 정리했습니다.

---

## 3층 구조

| 층 | 위치 | 쓰임 |
|---|---|---|
| 1. **실제 사례** | [`examples/my-canvas.md`](examples/my-canvas.md) | 실제로 채운 린캔버스 — 어떤 선택을, 어떤 근거로 |
| 2. **빈 템플릿** | [`template/lean-canvas.md`](template/lean-canvas.md) | 자기 사업을 투영하는 워크북 + 블록별 자문 질문 |
| 3. **의사결정 원칙** | [`docs/principles/`](docs/principles/) | 추출한 재사용 규율 (시급 방어선·플랫폼 집중도·부의 사다리) |

---

## 시작하기

1. [`examples/my-canvas.md`](examples/my-canvas.md)를 **먼저** 읽으세요. 추상 이론 전에 구체 사례를 봐야 감이 옵니다.
2. [`template/lean-canvas.md`](template/lean-canvas.md)를 복사해 자기 버전 초안을 채우세요.
3. [`docs/principles/`](docs/principles/) 의 원칙들을 자기 맥락에 맞게 숫자만 조정해 쓰세요.
4. 완성된 캔버스를 **한 장 그림**으로 시각화해 두세요 ([아래 시각화 가이드](#시각화-한-장으로-보기) 참조).

---

## 시각화 — 한 장으로 보기

텍스트 린캔버스는 정보 밀도는 높지만 **한눈에 안 들어옵니다**. 완성한 캔버스를 **한 장 이미지**로 만들어 두면:

- 매주 훑어보며 전략 드리프트를 감지하기 쉽다
- 미팅/피칭 때 첨부 자료로 바로 쓸 수 있다
- 동료·멘토에게 피드백 요청할 때 응답률이 높다

### 추천: Claude Code `diagram-design` 스킬

Claude Code에서:

```
/diagram-design
```

후 "내 린캔버스를 9블록 그리드 레이아웃으로 시각화해줘. 색상은 내 브랜드 톤에 맞춰서"라고 요청하면 **인라인 SVG HTML 파일** 한 장이 나옵니다. 편집도 텍스트로 가능합니다.

### 다른 옵션

| 도구 | 장점 | 단점 |
|---|---|---|
| **Notion 데이터베이스 보드** | 블록 편집·링크 연결 쉬움 | 이미지로 내보낼 때 레이아웃 깨짐 |
| **Excalidraw** | 손그림 톤, 자유로움 | Notion 미지원, 공유 시 별도 파일 |
| **Figma / Figjam** | 협업 피드백에 최적 | 1인 사업자에겐 과투자 |
| **공식 Lean Canvas** (leanstack.com) | 원저 양식 | 브랜딩 커스텀 어려움 |

> 개인 경험: Notion에 텍스트 SSOT + 분기마다 한 장짜리 PNG 내보내기 조합이 가장 유지비 낮았습니다.
