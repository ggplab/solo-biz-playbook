보안 사고 대응 — 6단계 프로세스

1인 사업자 공급망 사고 대응 — NIST 사이클을 6단계로 압축. 자동 감시(Detect)와 명시 대응(Triage)을 분리하고, 행동의 blast radius 순서를 지킨다.

다섯 가지 원칙

P1 — P5
P1

1차 소스 검증

뉴스·전달 메시지는 시작점이지 결론이 아니다. 최소 2개 독립 1차 소스 교차 확인 후에만 행동한다.

P2

미검증 도구 거부

처음 보는 npx·curl·pip 한 줄을 "긴급 점검"이라는 이유로 실행하지 않는다. 공격 자체가 그 모습으로 위장한다.

P3

IoC 기반 점검

패키지·파일명·고유 문자열·네트워크·외부 흔적 5종 IoC를 명시 명단화하고 스캔한다. 휴리스틱 금지.

P4

Blast Radius 작은 행동 우선

점검 → 격리 → 정화 → 회전 → 삭제 순서로만 진행. 증거 없이 회전·삭제로 점프하지 않는다.

P5

두 트랙 분리

자동 감시(launchd watchdog)와 명시 대응(subagent)을 섞지 않는다. 자동에 AI, 명시에 cron 모두 안티패턴.
