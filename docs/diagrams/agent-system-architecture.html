MODEL-NEUTRAL OPERATIONS

Agent-Neutral System Architecture

업무 지식을 특정 모델의 프롬프트 파일에 가두지 않고, 공용 문서와 실행 가능한 스크립트에 두는 구조입니다.

SSOT first

환경 지도와 유틸리티 레지스트리를 공용 문서로 두면 모델별 프롬프트 파일 사이의 드리프트를 줄입니다.

Adapters stay thin

Claude, Codex, Gemini용 파일은 “무엇을 읽고 무엇을 호출할지”만 담습니다. 업무 로직은 넣지 않습니다.

Portable automation

실행 로직이 스크립트에 있으면 새 에이전트를 붙일 때 포인터만 추가하면 됩니다.
