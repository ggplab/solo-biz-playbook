## 한줄 요약

"데이터 기반 의사결정 시스템을 AI로 설계하는 사람"이 직장인·1인 사업자·기업 실무팀에게 **end-to-end 자동화 설계 역량**을 전달하는 1인 교육 기업. **투자 모드**(수입 40h / IP 120h) 기본, **공수 효율 방어**가 원칙.

## Lean Canvas 9블록

### 1. Problem — 고객의 진짜 문제

**핵심 문제 2개**

1. **뭘 자동화할지 모른다**
   도구(n8n/Claude/GPT)는 들어봤는데, "내 업무 중 뭘 먼저 자동화해야 할지" 판단 기준이 없다. 도구 튜토리얼은 널려있지만 **상위 의사결정 프레임**이 없다.

2. **데이터가 흩어져 있다**
   매출·고객·운영 데이터가 엑셀·Notion·카톡·SaaS에 분산되어, "지금 내 사업/팀이 어떻게 돌아가는지" **숫자로 못 본다**. 그래서 자동화할 대상도, 개선 포인트도 보이지 않는다.

**두 문제의 연결 구조**
원인-결과 관계. **데이터가 흩어져 있으니 → 뭘 자동화할지 판단할 기준이 없다.** 지지플랩이 푸는 방식: 데이터를 한 곳에 모으고(Supabase) → 파이프라인으로 연결하고(n8n) → LLM으로 비정형 정제 → SQL 분석 → 의사결정 대시보드. 이 end-to-end 설계가 상품·콘텐츠의 뼈대.

> **배운 점**: 문제 후보 4~5개에서 2개로 줄이는 게 제일 어려웠다. "이것도 중요한데"를 버려야 포지셔닝이 선다. 나머지는 콘텐츠 앵글로만 다룬다.

### 2. Customer Segments — 고객 세그먼트

**전략 방향: B2C 주력 / B2B = 단기 캐시카우**

- B2B는 **캐시 + 레퍼런스 공급원** — 영구 메인 아님, B2C VOD 자산이 궤도에 오르기 전까지의 다리
- B2C VOD 체제가 월 반복수입 구조로 자리잡으면 B2B 비중을 점진적으로 낮춤 (단, 완전 포기는 아님 — 시급 방어선 유지 시 선별 수주)
- Nathan Barry 사다리 관점: 🪜 서비스(B2B) → 🏗️ 자산(B2C VOD·책·템플릿) 이동이 명시 목표

**우선순위 세그먼트**

| 우선 | 세그먼트 | 역할 | 현재 접점 |
|---|---|---|---|
| 🎯 **1차 (전략)** | **직장인 실무자** (데이터/마케팅/기획) | B2C VOD 핵심 구매자 | 온라인 교육 플랫폼 수강생, LinkedIn |
| 🎯 **1차 (전략)** | **1인 사업자/부업러** | B2C VOD + 템플릿 구매자, 커뮤니티 | Threads, 오픈채팅방 |
| 💰 **2차 (캐시카우)** | **기업 HRD/교육 담당자** | 단기 매출, 레퍼런스, 케이스 원재료 | 기업 강의 30회+, 플랫폼 경유 |
| 📣 부수 | 콘텐츠 크리에이터 | 브랜드 확산 앰플리파이어 | Threads |

**Early Adopter 프로필 (B2C VOD 첫 구매자)**
- 이미 n8n/LLM을 조금 써본 직장인·1인 사업자
- "도구 튜토리얼은 많이 봤는데 **뭘 자동화해야 할지, 데이터를 어디서부터 정리해야 할지 막막**" 한 사람
- 월 매출·업무 시간을 엑셀로라도 재고 있는 (또는 재고 싶어 하는) 사람
- **결정적 단서**: 오픈채팅방에서 "이런 거 자동화 가능한가요?" 질문하는 사람

**전략 파생 원칙**
- B2B에서 만든 교안·실습예제는 **B2C VOD의 원재료로 재활용** → 계약서 IP 소유권 조항 반드시 챙김
- B2B 수주 시 선호: B/C 유형 (반복 강의·강의만) > A 유형 (풀패키지) — 공수 방어 + IP 전환 가능성

> **배운 점**: B2B와 B2C 둘 다 "주력"이라고 잡으면 시간이 찢어진다. 하나는 전략 축, 하나는 캐시 축으로 역할 분리해야 지치지 않는다.

### 3. Unique Value Proposition — 고유 가치 제안

**메인 슬로건**
> "AI 시대, 혼자지만, 팀보다 강합니다"

**한 문장 정의**
> 도구 사용법이 아니라, **"데이터로 의사결정하는 시스템을 AI로 자동화하는 법"** 을 가르치는 교육자.

**하이컨셉 (pitch 한 줄)**
> "DB로 데이터 모으고 → 파이프라인 돌리고 → LLM으로 비정형 정제 → SQL로 분석 → 대시보드로 의사결정"의 **end-to-end를 혼자 설계할 수 있게 만드는 사람.**

**차별점 구조**
- 대부분의 AI 크리에이터: 도구 사용법에서 멈춤
- 대부분의 데이터 분석가: 자동화 못 함
- 지지플랩: **둘의 교차점 + 교육 전달력**

> **배운 점**: UVP를 "도구 이름"으로 쓰면 유행 따라 바뀐다. "데이터 의사결정"처럼 **도구보다 한 층 위**의 추상으로 잡아야 오래 간다.

### 4. Solution — 상품 라인업

Nathan Barry "Ladders of Wealth Creation" 4단 매핑:

| 계층 | 상품 | 가격 구조 | 비고 |
|---|---|---|---|
| 🪜 시간교환 | 1:1 멘토링, 컨설팅/자문 | 시간당 기본 단가 (상위 레인지) | 공수 큼 — 상한 관리 필요 |
| 🪜 서비스 | 기업 강의 (A/B/C 유형) | 시급제 | **B/C 유형 우선 협상** |
| 🪜 서비스 | VOD 제작 (기업 납품) | 완성 1h당 패키지 | 제작사 후반 부담 |
| 🪜 서비스 | 실습예제/커리큘럼 단독 납품 | 건당 | 이중 활용용 원재료 |
| 🏗️ 자산 | 온라인 강의 (자체) | 월 반복수입 목표 | **Q2 2026 런칭 예정** |
| 🏗️ 자산 | 저서 (기술 단행본) | 인세 | 2026 출간 예정 |
| 🏗️ 자산 | 템플릿·스타터킷 | 미정 | 강의/리드마그넷 이중활용 |
| ⚙️ 시스템 | 커뮤니티 | 현재 무료 | 유료 전환 검토 영역 |

**이메일 리스트 전략**
- 도구: 크리에이터 네이티브 뉴스레터 플랫폼 (한국 기준 메일리 계열)
- 방식: **콘텐츠-가치형 뉴스레터** — 별도 리드마그넷 제작 X. SNS 원재료를 심화·합본 형태로 이메일화
- 원재료: 업무일지, 시장 리서치, 강의/멘토링 반복 질문, 자동화 케이스
- SNS → 이메일 동선: SNS는 티저·요약본, 이메일은 풀버전·워크플로우 템플릿 포함
- **플랫폼 종속 완화**: 월 1회 구독자 명단 CSV 백업 정례화, Phase 2에서 자체 도구 이관 옵션 확보

> **배운 점**: 리드마그넷을 *전용으로* 만드는 건 ROI가 안 나왔다. 이미 쓴 콘텐츠를 심화 재포장하는 게 훨씬 싸게 먹힌다.

### 6. Revenue Streams — 수익 흐름

**현재 기본 운영 모드 (투자 모드)**
- 수입 활동 40h/월 → **매출 기본 베이스**
- IP 활동 120h/월 → 미래 반복 수입 (매출 기회비용은 베이스의 3배 상당 — 실질적으로 장기 자산 투자)

> 💡 절대 금액은 `docs/principles/06-invest-mode-math.md` (v1.1)에서 계산식과 함께 공개 예정. 여기서는 *비중·시간 배분·기회비용 개념*이 핵심.

**자산 비중 3개년 로드맵 (기본 시나리오)**

| 시점 | 자산 비중 | 상태 |
|---|---|---|
| 현재 (2026-04) | ~5% | 초기 (저서 출간·온라인 강의 런칭 직전) |
| +1년 | **25%** | 온라인 강의 2개 운영 + 인세 수입 시작 |
| +2년 | **40%** | VOD 3~4개 누적 + 저서 누적 인세 + 템플릿 수익 |
| +3년 | **55%** | 성숙기 문턱 — 시간 안 팔아도 매출 절반 |

**비중 목표 매트릭스**

| 유형 | 현재 | 1년 | 2년 | 3년 |
|---|---|---|---|---|
| 🪜 시간교환 (멘토링·컨설팅) | — | 15% 이하 | 10% 이하 | 5% 이하 |
| 🪜 서비스 (기업 강의·VOD 제작) | ~90% | 55~60% | 45~50% | 35~40% |
| 🏗️ **자산 (온라인 강의·인세·템플릿)** | ~5% | **25%** | **40%** | **55%** |
| ⚙️ 시스템 (유료 커뮤니티·구독) | 0% | 0~5% | 5~10% | 5~10% |

**전제 조건**
- IP 120h/월 유지 (투자 모드 기본값)
- 기업 강의 레퍼런스는 버리지 않음 — B/C 유형 선별 수주로 공수 방어하며 유지
- VOD 2~3개 누적 + 저서 롱테일 + 실습예제 템플릿화가 자산 축의 근간

**의사결정 방어선 (대시보드에서 확정)**
- 실투입 시급 하한 → 거절 (예외 없음) · 수치와 계산법: [`docs/principles/01-pricing-floor.md`](../docs/principles/01-pricing-floor.md)
- 단일 플랫폼 매출 비중 상한 → 초과 금지 · 근거: [`docs/principles/02-platform-concentration.md`](../docs/principles/02-platform-concentration.md)

> **배운 점**: 자산 비중을 **절대 매출 목표**가 아니라 **%** 로 관리하는 게 심리적으로 훨씬 낫다. 매출이 오르락내리락해도 구조 판단이 흔들리지 않는다.

### 7. Cost Structure — 비용 구조

**월 고정비 총액은 절대값 대신 구조로 공개**:

| 항목 | 비중 (전체 고정비 기준) | 비고 |
|---|---|---|
| 생활비 (집세·식비·교통·보험·개인지출) | **~93%** | 최우선 감축 검토 대상 (선택) |
| 사업 고정비 (외주·도구·세무·촬영) | **~7%** | 효율적 — 추가 압축 여지 작음 |

**운영 모드별 실현 가능성 재검증** (매출 배수 기준)

| 모드 | 매출 (베이스 배수) | 고정비 차감 후 | 실행 가능성 |
|---|---|---|---|
| 공격 (120h/40h) | 3.0x | 크게 +여유 | ✅ 가능 |
| **투자(기본) 40h/120h** | **1.0x** | **거의 손익분기** | ⚠️ 비상금 없이는 비상 대응 취약 |
| 최소 24h/136h | 0.6x | 🔴 적자 | ❌ **비상금 없이는 불가** |

**핵심 시사점**
1. "최소 모드"는 **비상금 보유가 전제**여야 실행 가능
2. "투자 모드"는 실제로는 **손익분기 모드** — 명칭과 체감의 괴리
3. **최우선 재무 과제: 비상금 6개월치 확보** — 없으면 IP 스프린트·장기 자산 투자가 구조적으로 불가능

**권장 재무 규율 (Profit First 기반)**
- 매출 입금 즉시 계좌 자동 분리: 세금 25% / 비상금 10%(목표 도달까지) / 재투자 10% / 생계 55%
- 비상금 목표 달성 후엔 비상금 비율을 **자산 투자(강의 장비·외주 편집 선투자)** 로 전환

> **배운 점**: 매출만 보고 "돈 벌었다"고 착각하지 말 것. 고정비 × 개월 수가 비상금 상한이고, 거기 도달 전까지는 실질 여유 0. 이걸 **배수로 보는 게 절대값보다 훨씬 정직**하다.

| 2 | **월 매출 중 자산 비중 %** | 월 | 1년 25% / 2년 40% / 3년 55% | 로드맵의 직접 측정치 |
| 3 | **IP 시간 확보율 (120h 대비)** | 주 | 80%+ | 로드맵 진도의 선행지표 |
| 4 | **B/C 유형 비중 (A 대비)** | 월 | 50%+ | 공수 방어 + IP 전환 가능성 |
| 5 | **비상금 적립 진도** | 월 | 6개월치 도달까지 지속 | 최소 모드 실행 가능성 확보 |

**Phase 2 추가 예정 (지금은 측정 불가)**
- 이메일 → 상품 구매 전환율 (뉴스레터·자체 온라인 강의 런칭 후)
- 오픈채팅방 → 유료 전환율 (커뮤니티 유료화 검토 시)
- SNS 팔로워 증가율 (이메일 구독자 선행지표로 더 중요해질 때)

> **배운 점**: KPI 10개 잡으면 아무도 안 본다. 5개로 줄이되, 각 지표가 **왜 중요한지** 문서에 박아두면 포기가 줄어든다.

---

### 9. Unfair Advantage — 대체 불가 자산

**쉽게 복제 안 되는 자산**
1. **저서 (기술 단행본 2026)** — 국내 n8n 단행본 선점 저자
2. **기업 강의 레퍼런스** — 대기업·커머스 플랫폼 30회+ 누적
3. **데이터 분석 근본 배경** — "AI 자동화만 아는" 크리에이터와 차별
4. **멘토링 400명+ 누적 피드백** — 실제 학습자 막힘 포인트 DB화 가능한 유일 자산
5. **커뮤니티 허브 운영자 포지션** — AI·자동화 오픈채팅
6. **데이터 커뮤니티 운영진 경력** — 도메인 내 신뢰 자본

**쉽게 복제되는 것 (Unfair Advantage 아님)**
- n8n 튜토리얼 콘텐츠 자체 (누구나 만듦)
- LLM 사용법 (레드오션)
- 특정 도구 스택 (유행 따라 바뀜)

> **핵심**: "도구 지식"이 아니라 "데이터 분석 × 자동화 × 교육"의 **교차점 + 실제 레퍼런스** 가 대체 불가 자산.

> **배운 점**: Unfair Advantage는 "내가 잘하는 것"이 아니라 "남이 따라하려면 몇 년 걸리는 것"이다. 프레임이 다르다.
